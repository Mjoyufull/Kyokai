# Statements {#stmt}

This section describes the semantics of Austral statements.

## Skip Statement {#stmt-skip}

The `skip` statement is a no-op.

Example:

```austral
skip;
```

## Let Statement {#stmt-let}

If `N` is an identifier, `T` is a type specifier, and `E` is an expression of
type `T`, then:

```austral
let N: T := E;
```

defines an immutable local variable.

The mutable form is:

```austral
var N: T := E;
```

which defines a mutable local variable.

A `let` statement is one of the few places where type information flows forward:
the declared type is used to disambiguate the type of the expression when the
expression is, for example, a call to a return-type polymorphic function.

## Let-destructure Statement {#stmt-let-destructure}

A let-destructure statement breaks apart a record, creating a binding for each
field.

If `R` is an expression of record type `T`, and `T` has fields
`{s_1: T_1, ..., s_n: T_n}`, then:

```austral
let {s_1: T_1, ..., s_n: T_n} := R;
```

destructures the record.

Bindings may be renamed:

```austral
let {s_i as n_i: T_i, ...} := R;
```

The parser also accepts `var { ... } := R;`; each destructured binding is then a
mutable local variable.

Examples:

```austral
let { symbol: Span[Nat8, Static], Z: Nat8, A: Nat8 } := isotope;
```

```austral
let { symbol: Span[Nat8, Static], Z as atomic_number: Nat8, A as mass_number: Nat8 } := isotope;
```

## Assignment Statement {#stmt-assign}

If `P` is an lvalue of type `T` and `E` is an expression of type `T`, then:

```austral
P := E;
```

stores the value of `E` in the location denoted by `P`.

Syntactically, an lvalue is a variable or a path starting from a variable.

## If Statement {#stmt-if}

If `{e_1, ..., e_n}` is a set of boolean expressions and
`{b_1, ..., b_n, b_else}` is a set of statements, then:

```austral
if e_1 then
    b_1;
else if e_2 then
    b_2;
...
else if e_n then
    b_n;
else
    b_else;
end if;
```

is the general form of an `if` statement.

Examples:

```austral
if test() then
    doSomething();
end if;
```

```austral
if test() then
    doSomething();
else
    doSomethingElse();
end if;
```

```austral
if a() then
    doA();
else if b() then
    doB();
else if c() then
    doC();
else
    doElse();
end if;
```

## Case Statement {#stmt-case}

If `E` is an expression of a union type with cases `{C_1, ..., C_n}`, then:

```austral
case E of
    when C_1(...) do
        B_1;
    ...
    when C_n(...) do
        B_n;
end case;
```

is a `case` statement.

Each union case must have a corresponding `when` clause, and each bound field in
a case must be named in the `when` clause.

Examples:

Suppose `Color` is the `RGB` / `Greyscale` union from the [union
definition](#union-definition) section. Then:

```austral
let c: Color := RGB(red => 10, green => 12, blue => 3);
case c of
    when RGB(red: Nat8, green: Nat8, blue: Nat8) do
        skip;
    when Greyscale(value: Nat8) do
        skip;
end case;
```

```austral
let o: Option[Int32] := Some(value => 10);
case o of
    when Some(value: Int32) do
        skip;
    when None do
        skip;
end case;
```

Bindings may be renamed:

```austral
case response of
    when Some(value as payload: Span[Nat8, Static]) do
        use(payload);
    when None do
        handleEmpty();
end case;
```

## While Loop {#stmt-while}

If `e` is an expression of type `Bool` and `b` is a statement, then:

```austral
while e do
    b;
end while;
```

is a while loop that iterates as long as `e` evaluates to `true`.

Example:

```austral
while true do
    doForever();
end while;
```

## For Loop {#stmt-for}

If `i` is an identifier, `s` and `f` are expressions compatible with `Index`,
and `b` is a statement, then:

```austral
for i from s to f do
    b;
end for;
```

is a for loop where `b` is executed once for each value of `i` in the interval
`[s, f]`, inclusive. The compiler typechecks `s` and `f` as values compatible
with `Index`, binds `i` as an immutable `Index`, initializes `i` to `s`, and the
C backend emits a loop condition equivalent to `i <= f` with `i++` after each
iteration.

Examples:

```austral
for i from 0 to n do
    doSomething(i);
end for;
```

```austral
for i from 0 to (argumentCount() - 1) do
    handleArg(i);
end for;
```

## Borrow Statement {#stmt-borrow}

If `x` is a variable of type `T`, `x'` is an identifier, `R` is a region name,
and `B` is a statement, then:

```austral
borrow x': &[T, R] := &x do
    B;
end borrow;
```

is a read-only borrow statement.

The mutable form is:

```austral
borrow x': &![T, R] := &!x do
    B;
end borrow;
```

and the reborrow form is:

```austral
borrow x': &![T, R2] := &~y do
    B;
end borrow;
```

where `y` is itself a mutable reference.

Within the body of the borrow statement, the renamed reference variable is
usable. For linear variables, the original borrowed variable may not be consumed
while it is borrowed. Mutable borrowing is rejected for immutable local
variables, and for parameters unless the parameter is itself a mutable
reference. Read-only borrow statements are also accepted for free values by the
current compiler.

Example:

```austral
borrow fooref: &[Foo, rho] := &foo do
    let subref: &[Nat8, rho] := &(fooref->bar);
    let b: Nat8 := !subref;
    Put_Character(b);
end borrow;
```

## Discarding Statement {#stmt-discard}

If `e` is an expression, then:

```austral
e;
```

evaluates the expression and discards its value.

Discarding statements are illegal when `e` has a linear type.

## Return Statement {#stmt-return}

If `e` is an expression, then:

```austral
return e;
```

returns from the function with value `e`.

`return` statements are illegal while there are unconsumed linear values still
in scope.
