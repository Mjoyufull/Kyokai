# Expressions {#expr}

This section describes the semantics of Austral expressions.

## Nil Constant {#expr-nil}

The expression:

```austral
nil
```

has type `Unit`.

## Boolean Constant {#expr-bool}

The Boolean constants are the identifiers `true` and `false`.

## Integer Constant {#expr-int}

Integer constants have type `Int32` by default, but to improve programmer
ergonomics, the compiler will try to find a type for an integer constant that
makes the surrounding context work. For example, if `x` has type `Nat8`, then
an expression like `x + 3` is valid and the `3` is interpreted to have type
`Nat8`.

## Float Constant {#expr-float}

Floating-point constants have type `Float64`.

## String Constant {#expr-str}

String constants have type `Span[Nat8, Static]`.

## Variable Expression {#expr-var}

An identifier that is not `nil`, `true`, or `false` denotes a local variable or
global constant, whose type is determined from the lexical environment.

## Arithmetic Expression {#expr-arith}

If `a` and `b` are expressions of the same integer or floating-point type `N`,
then:

```austral
a + b
a - b
a * b
a / b
```

are expressions of type `N`.

The surface arithmetic operators are typechecked as numeric operators and are
lowered to the corresponding `Austral.Pervasive.TrappingArithmetic` methods
(`trappingAdd`, `trappingSubtract`, `trappingMultiply`, and `trappingDivide`).
Therefore the operator is accepted only when a visible `TrappingArithmetic(N)`
instance can be resolved. In the shipped pervasive module this includes all
integer types, `Index`, `ByteSize`, and `Float64`; it does not include
`Float32`.

In the case of division, two rules apply:

- If `b` is zero, the program aborts due to a division-by-zero error.
- If `N` is a signed integer type, `a` is the minimum value of that type, and
  `b` is `-1`, the program aborts due to signed integer overflow.

## Function Call {#expr-funcall}

If `f` is the name of a function with parameters
`{p_1: T_1, ..., p_n: T_n}` and return type `R`, and we have expressions
`{e_1: T_1, ..., e_n: T_n}`, then:

```austral
f(e_1, ..., e_n)
```

and:

```austral
f(p_1 => e_1, ..., p_n => e_n)
```

are identical function-call expressions of type `R`.

## Method Call {#expr-methcall}

Let `m` be the name of a method in a typeclass. After
[instance resolution](#instance-resolution), the method has
parameters `{p_1: T_1, ..., p_n: T_n}` and return type `R`. Then:

```austral
m(e_1, ..., e_n)
```

and:

```austral
m(p_1 => e_1, ..., p_n => e_n)
```

are identical method-call expressions of type `R`.

## Record Constructor {#expr-rec-cons}

If `R` names a record type with slots `{s_1: T_1, ..., s_n: T_n}`, then:

```austral
R(s_1 => e_1, ..., s_n => e_n)
```

constructs a value of type `R`.

Positional arguments are also accepted when they appear in declaration order.

## Union Constructor {#expr-union-cons}

Let `U` be a union type with a case `C` having slots `{s_1: T_1, ..., s_n: T_n}`.
Then:

```austral
C(s_1 => e_1, ..., s_n => e_n)
```

constructs a value of `U` in case `C`.

Nullary cases are written with empty argument lists:

```austral
None()
```

## Cast Expression {#expr-cast}

The cast expression `e : T` is processed by the compiler roughly as follows:

1. **Literal clarification:** if `e` is an integer constant and `T` is an
   integer type (or `e` is a float constant and `T` is a float type), the
   constant is given type `T`.
2. **Reference downgrade:** if `e` has type `&![U, R]` and `T` is `&[U, R]`
   (same `U` and `R`), then `e : T` downgrades a write reference to a read
   reference.
3. **Return-type polymorphism:** if `e` is a call to a generic function or
   method and `T` fixes the otherwise-ambiguous result type, `e : T` supplies
   that type.
4. **General coercion:** otherwise, if `e`'s type can be matched to `T` under
   the usual typing rules, the cast is accepted as a coercion.

If `e` is an expression and `T` is a type, then:

```austral
e : T
```

is the expression that casts or constrains `e` to type `T`.

## Comparison Expression {#expr-comp}

If `a` and `b` are expressions of the same type, then:

```austral
a = b
a /= b
a < b
a <= b
a > b
a >= b
```

are comparison expressions of type `Bool` in the current compiler. The
typechecker currently enforces type equality between operands but does not call
the `TypeSystem.is_comparable` helper, so the accepted surface is broader than
the helper's intended comparable set. The C backend must still be able to render
the comparison for the resulting monomorphic type.

## Conjunction Expression {#expr-conj}

If `a` and `b` are Boolean-typed expressions, then `a and b` is the
short-circuiting conjunction operator and evaluates to a `Bool`.

## Disjunction Expression {#expr-disj}

If `a` and `b` are Boolean-typed expressions, then `a or b` is the
short-circuiting disjunction operator and evaluates to a `Bool`.

## Negation Expression {#expr-neg}

If `e` is a Boolean-typed expression, then:

```austral
not e
```

evaluates to the negation of `e`.

## If Expression {#expr-if}

If `c` is a Boolean-typed expression and `t` and `f` are expressions of the
same type `T`, then:

```austral
if c then t else f
```

is an `if` expression of type `T`.

## Path Expression {#expr-path}

A path is an identifier followed by one or more path elements. Each path element
is one of:

1. A slot accessor of the form `.s`.
2. A pointer slot accessor of the form `->s`.
3. An array or span indexing element of the form `[i]`.

Path expressions are used for:

1. Accessing fields and indexed elements.
2. Transforming references to records and spans into references to their
   contents.

Examples:

```austral
pos.x
star->pos.ra
stars[23]->distance
```

All paths must end in a free value unless the path is being used through an
explicit reference transform.

## Reference Path Expression {#expr-ref-path}

The form:

```austral
&(x->field->subfield)
```

is a reference-path transform. It is used to derive a reference to a nested
field or indexed component.

## Dereference Expression {#expr-deref}

If `e` is a reference to a value of type `T`, then:

```austral
!e
```

evaluates to the referenced value of type `T`.

## Borrow Expressions {#expr-borrow}

If `x` is a variable of type `T`, then:

```austral
&x
```

is an anonymous read-only borrow expression of type `&[T, R]`, where `R` is a
fresh unnamed region. The current compiler accepts read-only borrows of both
free and linear values.

Similarly:

```austral
&!x
```

is an anonymous mutable borrow of type `&![T, R]`.

If `y` is already a mutable reference, then:

```austral
&~y
```

is a reborrow expression that produces a fresh mutable reference to the same
underlying value.

Borrow expressions cannot escape the statement in which they appear.

## sizeof Expression {#expr-sizeof}

If `T` is a type specifier, then:

```austral
sizeof(T)
```

is an expression of type `ByteSize` that evaluates to the size of `T` in bytes.
