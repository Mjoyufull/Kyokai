# Declarations {#decl}

This section describes the kind of declarations that can appear in Austral
modules.

## Opaque Constant {#opaque-constant}

If `c` is an identifier and `T` is a type specifier, then:

```austral
constant c : T;
```

is an opaque constant declaration.

These can only appear in the module interface file, and must be accompanied by a
matching constant definition in the module body file.

Example:

```austral
module Example is
    constant pi : Float64;
end module.

module body Example is
    constant pi : Float64 := 3.14;
end module body.
```

## Constant Definition {#constant-definition}

If `c` is an identifier, `T` is a type specifier, and `V` is a constant
expression of type `T`, then:

```austral
constant c : T := V;
```

defines a constant named `c` of type `T` with value `V`.

Constant definitions can only appear in the module body file.

If the module interface file has a corresponding opaque constant declaration,
then the constant is public and can be imported by other modules. Otherwise, it
is private.

## Opaque Type Declaration {#opaque-type-declaration}

If `T` is an identifier and `U` is a universe marker, then:

```austral
type T: U;
```

is an opaque type declaration. It can only appear in a module interface file.
The module body must provide a matching concrete record or union definition for
the type name.

## Record Definition {#record-definition}

A record is a collection of named fields.

If `R` is an identifier, `U` is a universe, and `{s_1: T_1, ..., s_n: T_n}` is
a set of slots, then:

```austral
record R: U is
    s_1: T_1;
    ...
    s_n: T_n;
end;
```

defines the record `R`.

Examples:

```austral
record Vector3: Free is
    x: Float32;
    y: Float32;
    z: Float32;
end;
```

Construction can use positional or named arguments:

```austral
let v1: Vector3 := Vector3(0.0, 0.0, 0.0);
let v2: Vector3 := Vector3(
    x => 0.0,
    y => 0.0,
    z => 0.0
);
```

## Union Definition {#union-definition}

Unions are tagged sums. Each case is either nullary or has a set of named
fields.

If `U` is an identifier, `K` is a universe, and the cases are `{C_1, ..., C_n}`,
then:

```austral
union U: K is
    case C_1;
    case C_2 is
        value: T;
    case C_3 is
        left: L;
        right: R;
end;
```

defines the union `U`.

Examples:

```austral
union Option[T: Type]: Type is
    case None;
    case Some is
        value: T;
end;
```

```austral
union Color: Free is
    case RGB is
        red: Nat8;
        green: Nat8;
        blue: Nat8;
    case Greyscale is
        value: Nat8;
end;
```

Construction examples:

```austral
let o1: Option[Int32] := None();
let o2: Option[Int32] := Some(value => 10);
let c1: Color := RGB(red => 10, green => 12, blue => 3);
let c2: Color := Greyscale(value => 50);
```

## Function Declaration {#function-declaration}

Let `f` be an identifier, `{p_1: T_1, ..., p_n: T_n}` a set of value
parameters, and `R` a return type. Then:

```austral
function f(p_1: T_1, ..., p_n: T_n): R;
```

declares a concrete function.

More generally, given type parameters `{tp_1: K_1, ..., tp_n: K_n}`:

```austral
generic [tp_1: K_1, ..., tp_n: K_n]
function f(p_1: T_1, ..., p_n: T_n): R;
```

declares a generic function.

Function declarations can only appear in the module interface file, and there
must be a corresponding function definition in the module body file with the
same signature.

Example:

```austral
module Example is
    generic [T: Type]
    function identity(x: T): T;
end module.

module body Example is
    generic [T: Type]
    function identity(x: T): T is
        return x;
    end;
end module body.
```

## Function Definition {#function-definition}

If `f` is an identifier, `{p_1: T_1, ..., p_n: T_n}` is a set of value
parameters, `R` is a return type, and `S` is a statement, then:

```austral
function f(p_1: T_1, ..., p_n: T_n): R is
    S
end;
```

defines a concrete function.

More generally:

```austral
generic [tp_1: K_1, ..., tp_n: K_n]
function f(p_1: T_1, ..., p_n: T_n): R is
    S
end;
```

defines a generic function.

If there is a corresponding declaration in the module interface file, the
function is public. Otherwise, it is private to the module body.

## Typeclass Definition {#typeclass-definition}

Given a typeclass name `TC`, a type parameter `T: U`, and a set of method
signatures, then:

```austral
typeclass TC(T: U) is
    method m_1(...): R_1;
    ...
    method m_n(...): R_n;
end;
```

defines the typeclass `TC`.

Typeclass definitions may appear in the module interface file or in the module
body file.

Example:

```austral
typeclass Printable(T: Free) is
    method print(value: T): Unit;
end;
```

## Instance Declaration {#instance-declaration}

If `TC` is the name of a typeclass and `T` is a type specifier, then:

```austral
instance TC(T);
```

declares an instance of `TC` for `T`.

Instance declarations can only appear in the module interface file, and must
have a matching instance definition in the module body file.

## Instance Definition {#instance-definition}

If `TC` names a typeclass, `T` is a type in the corresponding universe, and the
methods required by `TC` are implemented, then:

```austral
instance TC(T) is
    method m_1(...): R_1 is
        S_1
    end;
    ...
    method m_n(...): R_n is
        S_n
    end;
end;
```

defines a concrete instance.

More generally:

```austral
generic [tp_1: K_1, ..., tp_n: K_n]
instance TC(T) is
    ...
end;
```

defines a generic instance.

Example:

```austral
typeclass Printable(T: Free) is
    method print(value: T): Unit;
end;

instance Printable(Int32) is
    method print(value: Int32): Unit is
        printLn(value);
        return nil;
    end;
end;
```
